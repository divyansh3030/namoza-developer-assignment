# Task 1 – GTM & GA4 Event Tracking Strategy

## Overview

This task demonstrates a practical implementation of a Google Tag Manager (GTM) and Google Analytics 4 (GA4) tracking strategy for a healthcare appointment booking journey.

In addition to the required event schema and dataLayer examples, I implemented a working frontend demonstration to simulate how a frontend developer would emit custom events for GTM to consume.

---

- Design a scalable GTM event schema
- Implement custom dataLayer events
- Simulate a multi-step booking flow
- Define a GA4 conversion funnel
- Recommend a Google Ads conversion event

## Event Flow

```
booking_started
        ↓
booking_step_1_complete
        ↓
booking_step_2_complete
        ↓
booking_step_3_complete
        ↓
booking_completed
```

---

## Frontend Implementation

A key part of this task was demonstrating how custom dataLayer events are generated by the frontend application.

Each booking step triggers a dedicated:

```javascript
window.dataLayer.push(...)
```

Google Tag Manager listens for these custom events and forwards them to Google Analytics 4.

This approach enables accurate tracking of multi-step booking funnels, which GTM cannot detect automatically.

---

## Google Ads Recommendation

Recommended Conversion:

**booking_completed**

Reason:

This event represents a successfully confirmed appointment and therefore serves as the primary business conversion for campaign optimization.
